<p>The CDA Advance Healthcare Directive Interoperability (ADI) implementation guide (IG) is a companion guide to the FHIR ADI IG covering this same scope. The FHIR ADI IG explains how to represent a person’s goals, preferences and priorities for medical treatment and interventions regarding future medical care, and how to represent a person's consent to delegate medical decision-making to a healthcare agent if they are not able to communicate and make decisions for themself. The FHIR ADI IG also explains how to represent directives for life-sustaining treatments a patient or their representative has decided the patient does or doesn't want to receive and a provider has ordered in advance. In addition to document content representation, The FHIR ADI IG covers the use of FHIR Document Managemnt functionality to facilitate version management, exchange, and verification of document status and authenticity.</p>

<p>The CDA ADI IG provides the data structures (templates) used in CDA to represent the same types of advance healthcare directive documents and structured data as covered by the FHIR ADI IG. ### Introduction

### Background

### Relationships to Other IGs

This implementation guide (IG) contains CDA templates for the Personal Advance Care Plan (ACP) and Portable Medical Order (PMO) document types defined in the FHIR Advance Healthcare Directives Interoperability IG. For detailed information regarding the use cases supported for these document types and additional background on the content contained in each go to https://build.fhir.org/ig/HL7/pacio-adi/index.html
